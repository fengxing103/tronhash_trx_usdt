  • TRX参与数量只支持10-100000数字币。
  • USDT参与数量只支持10-10000数字币。
  • 低于最低金额时，不可参与抽奖且参与金额不进行退回。

中奖规则

  • 转账金额的个位数为0～4，代表下注小，区块哈希值(Block hash)后三位中最右侧的数字（只看数字，不看字母）也为0～4；或转账金额的个位数为5～9，代表下注大，区块哈希值(Block hash)后三位中最右侧的数字（只看数字，不看字母）也为5～9时，则成功中奖。（注：hash值后三位全为字母时，不中奖）
  • 中奖后系统5秒内自动返还转账金额1.98倍的同类数字币。(由于区块链技术为了保证资金安全，每笔交易都需记录，防篡改，且全球使用数字币人口庞大，故5秒转账后可能会出现些许延迟到账情况，敬请耐心等待)
  • 转账金额的个位数和区块哈希值(Block hash)后三位中最右侧的数字（只看数字，不看字母）同为单数或同为双数时，则成功中奖。（注：hash值后三位全为字母时，不中奖）
  • 中奖后系统5秒内自动返还转账金额1.98倍的同类数字币。(由于区块链技术为了保证资金安全，每笔交易都需记录，防篡改，且全球使用数字币人口庞大，故5秒转账后可能会出现些许延迟到账情况，敬请耐心等待)
  • 转账后区块哈希值(Block hash)最后的两位为“数字+字母”或“字母+数字”组合时，则成功中奖。
  • 中奖后系统10秒内自动返还转账金额1.98倍的同类数字币。(由于区块链技术为了保证资金安全，每笔交易都需记录，防篡改，且全球使用数字币人口庞大，故10秒转账后可能会出现些许延迟到账情况，敬请耐心等待)
  • 采用【区块哈希值】后五位。平台点数：后五位中的前3位之和；玩家点数：后五位中的后3位之和；字母=10点
  • 根据牛几的几倍倍数结算（牛一*1倍….牛牛*10倍）
  • 转账金额=投注金额*10倍 如投注100需转账1000
  • 赔率：本金+投注额*牛几倍数*0.95
  • 特殊规则 1、牛一~牛二 玩家与平台同点 平台胜；2、牛三~牛牛 玩家与平台同点，按各自结果中的最数字依次比较，字母=10；3、牛三~牛牛 玩家与平台同点，各自结果中数字同点，和局，原额退还给玩家。

  • Q1：什么是区块哈希值(Block hash)？

    区块哈希值是一段数据的DNA，每个区块哈希值都是唯一的、随机的且不可破解的。 点击查看更多介绍

  • Q2：什么是区块链？

    区块链从本质上讲，它是一个共享数据库，存储于其中的数据或信息。具有不可伪造、全程留痕、可以追溯、公开透明、集体维护等特征。
    点击查看更多介绍

  • Q3：中心化钱包？去中心化钱包？为什么中心化钱包无法查询转账？

    举一个例子：您自己的钱包地址是A，您给地址E转入10个USDT，但您用交易哈希值去查询，只能查询到服务商随机某个账本地址B（或C或D等等）与E的转账。同理E也无法得知这10个USDT是否是您转出的，因为E只能看见是账本地址B给他转入10个USDT。这就是为什么中心化钱包无法在区块链上查询真实转账信息。
    点击查看更多介绍

  • Q4：为什么我会连续中奖或不中奖？

    首先连续多次不中奖的是小概率事件（小概率事件定义），其次中奖与不中奖是随机事件，随机事件的发生的频率会趋于一个稳定值。在非迭代的情况下，事物出现的概率对每个人来讲是相等的，这个情况下，每个人都是公平的。同时随机事件会遵大数定律（在试验不变的条件下，重复试验多次，随机事件的频率近似于它的概率大数定律定义），在转账频次不断增加的情况下，中奖与不中奖的概率会趋于一个稳定值。所以请您继续转账，随着转账频次增加则中奖总概率会提升。

  • Q5：为什么会延迟到帐？

    由于区块链技术为了保证资金的安全性，会进行多次核算，比对，确认。所以计算速度较慢，但是贵在安全。

  • Q6：为什么转币时有那么多USDT的类型选项？我该选哪一个？

    全球有许多知名公链（点击查看全球知名公链简介）都支持进行USDT转账，而平台是基于波场TRC20技术。所以平台的转账地址是属于USDT-TRC20， 您应该选择USDT-TRC20；如果您选择了其他类型，平台则无法收到您的USDT，这会造成您和平台双方的损失。
